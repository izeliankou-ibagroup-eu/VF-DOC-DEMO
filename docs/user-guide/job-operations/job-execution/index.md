# Job Execution

Push the *Play* button ![](../../../assets/user-guide-img/image47.jpeg) to run the job:

Its status changes from *Draft* to *Pending* ![](../../../assets/user-guide-img/image57.jpeg)

Push Refresh to update the status. 
It should turn to *Running* ![](../../../assets/user-guide-img/image56.png)

While running, it can be interrupted with the *Stop* button. ![](../../../assets/user-guide-img/image79.jpeg)
When a job is completed, its status is
*Succeeded* or *Failed*.
Use the Logs button ![](../../../assets/user-guide-img/image50.png) to analyze job logs. It gets you to Logs screen:

![](../../../assets/user-guide-img/image58.jpeg)


Logs can be filtered by level. Such as:

* INFO
* WARNING
* ERROR
* DEBUG
* RESULT

You can also view earlier job runs data with the *Job History* button:

![](../../../assets/user-guide-img/image59.png)

It takes you to *Job History* screen, containing each job run data, including logs:

![](../../../assets/user-guide-img/image60.png)
